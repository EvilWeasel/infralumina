# Plan Archive

Zweck:
- Archiv fuer abgeschlossene Plan-Segmente aus `current-plan.md`.
- Nachvollziehbarer Verlauf ohne den aktiven Kontext aufzublaehen.

Regeln:
- Nur abgeschlossene Segmente eintragen.
- Ein Archivblock pro abgeschlossenem Work Item.
- Reihenfolge chronologisch (neueste oben).
